map与parseInt

技术,js

[‘1’,’2’,’3’].map(parseInt)结果是什么？

在闲逛面试题的时候偶然看到了这道题

在解决这个问题前我们首先应该了解他所用到东西

也就是map和parseInt

当然实践出真知，我们可以先行打开控制台看一下这个结果

为什么会变成这个样子呢？

现在让我们回头来看看map和parseInt吧

map

首先来看看MDN上是怎么说的吧

map()方法创建一个新数组，其结果是该数组中的每个元素都调用一个提供的函数后返回的结果。

他的语法是这样的

var new_array = arr.map(function callback(currentValue[, index[, array]]) {
// Return element for new_array
}[, thisArg])

其中callback方法里面有三个参数

  • currentValue, 是callback 数组中正在处理的当前元素。

  • index可选, 是callback 数组中正在处理的当前元素的索引。

  • array可选, 是callback map 方法被调用的数组。

通常我们只用到callback方法里面的currentValue这个参数

而map的返回值为一个新数组，其中的每个元素都是callback函数的结果

也就是这个数组[‘1’,’2’,’3’]去执行map方法时，在对其中的’1’,’2’,’3’进行遍历处理，此处是调用parseInt方法，最后返回一个新的数组

那么parseInt又是个什么方法呢？

parseInt

同样让我们来看看MDN的说法

parseInt(string, radix) string为字符串，radix为介于2-36之间的数。使用者告诉这个函数string（比如11）是radix（比如2）进制的，函数将固定返回string以十进制时显示的数（3）

语法如下

parseInt(string, radix);

很明显这个方法是需要传递两个参数的

string是要被解析的值。如果参数不是一个字符串，则将其转换为字符串(使用 ToString 抽象操作)。字符串开头的空白符将会被忽略。

radix一个介于2和36之间的整数(数学系统的基础)，表示上述字符串的基数。比如参数”10”表示使用我们通常使用的十进制数值系统。始终指定此参数可以消除阅读该代码时的困惑并且保证转换结果可预测。当未指定基数时，不同的实现会产生不同的结果，通常将值默认为10。

而他的返回值是返回解析后的整数值。 如果被解析参数的第一个字符无法被转化成数值类型，则返回NaN。

当radix参数为n时，将会把第一个参数看作是一个数的n进制表示，而返回的值则是十进制的。例如：

那么如果radix参数为空呢？

在基数为undefined,或者基数为 0,或者没有指定的情况下，JavaScript 会作如下处理：

  • 如果字符串 string 以”0x”或者”0X”开头, 则基数是16 (16进制).
  • 如果字符串 string 以”0”开头, 基数是8（八进制）或者10（十进制），那么具体是哪个基数由实现环境决定。ECMAScript 5 规定使用10，但是并不是所有的浏览器都遵循这个规定。因此，我们使用时最好都要明确给出radix参数的值。
  • 如果字符串 string 以其它任何值开头，则基数是10 (十进制)。
